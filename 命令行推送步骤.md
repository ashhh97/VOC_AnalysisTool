# 通过命令行推送代码到GitHub

## 仓库地址
https://github.com/ashhh97/VOC_AnalysisTool.git

## 前置条件：安装Xcode命令行工具

**必须步骤：** Git需要Xcode命令行工具才能运行

```bash
xcode-select --install
```

安装步骤：
1. 运行上面的命令
2. 在弹出的对话框中点击"安装"
3. 等待安装完成（可能需要10-30分钟）
4. 安装完成后，重启终端

## 推送步骤

### 方法1：使用自动化脚本（推荐）

安装完Xcode命令行工具后，运行：

```bash
cd /Users/SL/Desktop/voc-analysis-tool
./push-to-github.sh
```

### 方法2：手动执行命令

如果脚本不工作，可以手动执行以下命令：

```bash
# 1. 进入项目目录
cd /Users/SL/Desktop/voc-analysis-tool

# 2. 初始化Git仓库（如果还没有）
git init

# 3. 添加远程仓库
git remote add origin https://github.com/ashhh97/VOC_AnalysisTool.git
# 如果已经存在，使用：git remote set-url origin https://github.com/ashhh97/VOC_AnalysisTool.git

# 4. 添加所有文件
git add .

# 5. 创建提交
git commit -m "Initial commit: VOC分析工具"

# 6. 创建并切换到main分支（如果还没有）
git checkout -b main

# 7. 推送到GitHub
git push -u origin main
```

**如果main分支不行，尝试master分支：**

```bash
git checkout -b master
git push -u origin master
```

## 常见问题

### 1. 如果提示需要身份验证

GitHub现在要求使用Personal Access Token而不是密码：

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 设置权限：勾选 `repo` 权限
4. 生成token并复制
5. 推送时，用户名输入您的GitHub用户名，密码输入刚才生成的token

### 2. 如果提示 "remote: Support for password authentication was removed"

需要配置SSH密钥或使用Personal Access Token（见上面）

### 3. 如果Git命令找不到

确保Xcode命令行工具已安装完成，然后重启终端
